# Outward-Facing Package

Week 13 turns the repository into something that can be shown quickly and coherently.

## Working Rule

Use `make outward-package` or `python -m spotify.outward_package` to assemble a share-ready package from already-generated artifacts.

The package should include:

- one flagship claim-to-demo review pack
- one canonical Taste OS showcase
- one current control-room review sample
- one creator-intelligence strategy brief
- one safety / research showcase artifact
- one concise summary of the four primary branches

## Review Order

1. Claim-to-demo pack for the fastest external story
2. Taste OS for the product story
3. Control Room for the operating story
4. Creator Intelligence for the external strategy story
5. Safety and Research Platform for the infrastructure and publication story

## Honesty Rule

The package should preserve the current state of the system, including any research or benchmark gaps. It is a polished presentation layer, not a marketing rewrite.
