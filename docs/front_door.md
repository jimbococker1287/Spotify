# Front Door

The front door is the external-review surface for the repo. It turns the strongest product demo, the strongest research claim, and the four-branch portfolio into one file that can be opened first.

Use:

```bash
make front-door
```

Outputs:

- `outputs/analysis/front_door/index.html`
- `outputs/analysis/front_door/front_door.md`
- `outputs/analysis/front_door/front_door.json`

Review order:

1. Open the landing page.
2. Follow the claim-to-demo pack.
3. Open the branch portfolio if you want the repo-wide map.

The landing page should stay lightweight and opinionated. It is not meant to replace the deeper markdown packs; it is meant to make the first five minutes clear.
